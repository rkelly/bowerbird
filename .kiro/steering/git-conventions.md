# Git Conventions

- Make atomic commits after each logical unit of work (a fix, a new file, a spec document, a refactor step, etc.)
- Use conventional commit prefixes: `fix:`, `feat:`, `docs:`, `refactor:`, `chore:`, `test:`
- Keep commits small and focused — one concern per commit
- Do not batch unrelated changes into a single commit
